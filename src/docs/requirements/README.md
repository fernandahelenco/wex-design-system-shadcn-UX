# Requirements

This folder contains **product + UX requirements** that guide design and implementation.

## How to use this

- Start in `global/` to understand the project intent and shared constraints.
- Use module folders (e.g. `homepage/`) for **MVP requirements**, flows, edge cases, copy, and API contracts scoped to that module.
- Keep these documents **implementation-adjacent**: specific, testable, and easy to update.

## Conventions

### Status block (keep current)

Each doc should include:
- **Owner**
- **Last updated**
- **Status**: draft | review | approved
- **Applies to**: global | <module>

### MVP-first

Put the MVP decisions and acceptance criteria near the top. Move background and future ideas lower.
